# Contributing to GPARS

Thank you for your interest in contributing to the General-Purpose Agent Reference Standard.

## How to Contribute

### Feedback and Discussion

- Open an [issue](../../issues) to ask questions, report problems, or suggest improvements.
- Use clear titles that describe the topic (e.g., "Clarify EMO boundary for cached memory" not "Question about spec").

### Proposing Changes

1. Fork the repository.
2. Create a branch for your change.
3. Make your edits. If modifying the spec, ensure consistency with existing terminology and conformance language.
4. Submit a pull request with a clear description of what changed and why.

### What We're Looking For

- Corrections to ambiguities or inconsistencies in the spec.
- Additional edge cases for the EMO classification.
- Real-world implementation feedback.
- Example manifests for different agent types.

### What's Out of Scope for v0.1

See [Non-Goals](./spec/v0.1/gpars_v0.1_spec.md#15-non-goals-v01) in the spec. Proposals for identity infrastructure, authentication protocols, or policy rule syntax are welcome as discussion topics for future versions but will not be merged into v0.1.

## Style Guidelines

- Use RFC 2119 conformance language (MUST, SHOULD, MAY) only in normative sections.
- Keep terminology consistent with Section 4 of the spec.
- Non-normative content (examples, rationale, appendices) should be clearly marked.
